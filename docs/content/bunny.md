# {{< icon "fa fa-cloud" >}} Bunny.net Storage

[Bunny.net](https://bunny.net/) is a content delivery platform that includes
edge storage. Bunny Storage provides a simple HTTP-based object storage API
with storage zones replicated across multiple regions.

## Configuration

Here is an example of making a bunny remote.

First, create a [Bunny.net](https://bunny.net/) account and set up a
Storage Zone in the Bunny dashboard. You will need the **storage zone name**
and its **API key** (found under FTP & API Access in the storage zone settings).

Now run:

    rclone config

This will guide you through an interactive setup process:


Once configured you can use it like this:

List directories in the top level of your storage zone:

    rclone lsd mybunny:

List all files in the storage zone:

    rclone ls mybunny:

Upload a local directory:

    rclone copy /home/source mybunny:backup

### Modification times

Bunny Storage does not support setting modification times on objects.
The modification time reported by rclone is the upload time recorded by
Bunny's servers.

### Checksums

Bunny Storage provides SHA256 checksums for all stored objects. These
checksums are computed server-side on upload and returned in directory
listings. Rclone uses them for integrity verification during transfers.

You can verify checksums with:

    rclone check /local/path mybunny:remote/path --checksum

### MIME types

The Bunny Storage API includes a `ContentType` field in its directory
listing JSON responses, and accepts a `Content-Type` header on upload.
However, the storage API does not currently populate the `ContentType`
field in listing responses regardless of what `Content-Type` header is
sent during upload -- the field is always returned as an empty string.
Because rclone cannot read back the MIME type that was written, MIME
type support is disabled (`ReadMimeType` and `WriteMimeType` are both
`false`). If Bunny enables this in a future API revision, support can
be turned on by setting both feature flags to `true`.

### Empty directories

Bunny Storage supports creating empty directories, but its listing API
returns the same response (an empty JSON array) for both an empty
directory and a non-existent path. Because rclone cannot distinguish
between the two cases, `CanHaveEmptyDirectories` is set to `false`.

### Regions

Bunny Storage zones can be created in different regions. Select the
endpoint matching the region your storage zone is in:

| Endpoint | Region |
|---|---|
| `storage.bunnycdn.com` | Frankfurt, DE (default) |
| `uk.storage.bunnycdn.com` | London, UK |
| `ny.storage.bunnycdn.com` | New York, US |
| `la.storage.bunnycdn.com` | Los Angeles, US |
| `sg.storage.bunnycdn.com` | Singapore, SG |
| `se.storage.bunnycdn.com` | Stockholm, SE |
| `br.storage.bunnycdn.com` | São Paulo, BR |
| `jh.storage.bunnycdn.com` | Johannesburg, SA |

### Standard options

Here are the Standard options specific to bunny (Bunny.net Storage).

#### --bunny-storage-zone

Storage zone name.

Properties:

- Config:      storage_zone
- Env Var:     RCLONE_BUNNY_STORAGE_ZONE
- Type:        string
- Required:    true

#### --bunny-access-key

Storage zone API key (password).

**NB** Input to this must be obscured - see [rclone obscure](/commands/rclone_obscure/).

Properties:

- Config:      access_key
- Env Var:     RCLONE_BUNNY_ACCESS_KEY
- Type:        string
- Required:    true

## Limitations

- Modification times cannot be set; the time shown is the server-side upload time
- No `Move` or `Copy` server-side operations; rclone will download and re-upload
- No `Purge`; rclone will delete files individually
- `rclone about` is not supported
- MIME types are not preserved (see [MIME types](#mime-types) above)
