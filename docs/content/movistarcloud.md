# {{< icon "fa fa-cloud" >}} Movistar Cloud

[Movistar Cloud](https://micloud.movistar.es) (MiCloud) is a cloud storage
service provided by Telefónica for Movistar customers in Spain. It offers file
and photo storage accessible via web and mobile apps.

## Configuration

Movistar Cloud uses cookie-based authentication. You will need to obtain a
`JSESSIONID` cookie from a logged-in browser session.

### Obtaining the JSESSIONID cookie

1. Go to [https://micloud.movistar.es](https://micloud.movistar.es) in your
   browser and log in with your Movistar credentials.
2. Open the browser developer tools (F12 or right-click → Inspect).
3. Go to the **Application** tab (Chrome) or **Storage** tab (Firefox).
4. Under **Cookies**, find the cookie named `JSESSIONID` for the domain
   `micloud.movistar.es`.
5. Copy its value.

### Running rclone config

Run `rclone config` and follow the prompts:

### Session expiry

The JSESSIONID is a session cookie and will expire. When it does, you will
need to log in again in your browser and update the value:

    rclone config update movistarcloud jsessionid NEW_JSESSIONID_VALUE

## Usage

List directories in the root:

    rclone lsd movistarcloud:

List all files:

    rclone ls movistarcloud:

Copy a local directory to Movistar Cloud:

    rclone copy /local/path movistarcloud:remote/path

Copy files from Movistar Cloud to local:

    rclone copy movistarcloud:remote/path /local/path

## Limitations

- **No hash support**: The API does not expose file hashes, so
  `--checksum` based syncing is not available.
- **No server-side copy or move**: Files are always downloaded and
  re-uploaded for copy/move operations.
- **No SetModTime**: Modification times cannot be updated on existing
  files. Uploads preserve the modification time at upload time.
- **Session-based auth**: The JSESSIONID cookie expires and must be
  refreshed manually.
