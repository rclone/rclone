# {{< icon "fa fa-cloud" >}} Movistar Cloud

[Movistar Cloud](https://micloud.movistar.es) (MiCloud) is a cloud storage
service provided by Telefónica for Movistar customers in Spain. It offers file
and photo storage accessible via web and mobile apps.

## Configuration

Movistar Cloud authenticates using the same OAuth SMS one-time-password (OTP)
login flow as the official apps. You do **not** need to extract any cookies
from your browser. During configuration you provide the phone number
associated with your Movistar Cloud account, Movistar sends you an SMS with a
one-time code, and you type that code into rclone. rclone then stores the
resulting credentials and refreshes them automatically from then on, so you
should not need to log in again.

### Running rclone config

Run `rclone config` and follow the prompts. When prompted, enter the phone
number for your Movistar Cloud account without the country code (for example
`612345678` for a Spanish mobile). A leading `+34`/`0034`/`34` country code is
accepted too and stripped automatically:

```text
No remotes found, make a new one?
n) New remote
d) Delete remote
c) Copy remote
s) Set configuration password
q) Quit config
n/d/c/s/q> n

Enter name for new remote.
name> movistarcloud

Option Storage.
Type of storage to configure.
Choose a number from below, or type in your own value.
...
XX / Movistar Cloud
   \ (movistarcloud)
...
Storage> movistarcloud

Option phone_number.
Phone number associated with your Movistar Cloud account, without the country code (e.g. 612345678).
Enter a value.
phone_number> 612345678

Edit advanced config?
y) Yes
n) No (default)
y/n> n
```

At this point rclone starts the login and sends a one-time code via SMS, then
prompts you to enter it:

```text
An OTP code has been sent via SMS to 612345678.

Option config_otp.
Enter the OTP code sent to your phone via SMS
Enter a value.
config_otp> 123456
```

Open the SMS on the phone you configured, enter the code you received, and
rclone completes the configuration:

```text
Authentication successful!

Configuration complete.
Options:
- type: movistarcloud
- phone_number: 612345678
- access_token: XXXX
- jsessionid: XXXX
- device_id: XXXX
Keep this "movistarcloud" remote?
y) Yes this is OK (default)
e) Edit this remote
d) Delete this remote
y/e/d> y
```

The `access_token`, `jsessionid` and `device_id` are stored automatically and
you should not need to edit them by hand.

### Session refresh

After the one-time SMS OTP login, rclone keeps the session alive on its own.
When the session expires, rclone automatically uses the stored access token to
fetch fresh credentials and saves them back to the config file, so in normal
use you should not have to repeat the SMS login.

If the stored credentials ever become invalid (for example if the access is
revoked from the Movistar side), simply reconfigure the remote to run the SMS
OTP login again:

    rclone config reconnect movistarcloud:

## Usage

List directories in the root:

    rclone lsd movistarcloud:

List all files:

    rclone ls movistarcloud:

Copy a local directory to Movistar Cloud:

    rclone copy /local/path movistarcloud:remote/path

Copy files from Movistar Cloud to local:

    rclone copy movistarcloud:remote/path /local/path

## Limitations

- **No hash support**: The API does not expose file hashes, so
  `--checksum` based syncing is not available.
- **No server-side copy**: Files are always downloaded and
  re-uploaded for copy operations. Server-side move is supported.
- **SMS login required**: Initial setup requires access to the phone
  number on the account to receive the one-time login code (OTP). After
  that, rclone refreshes the session automatically.
