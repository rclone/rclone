# Databricks Unity Catalog

The `databricks` backend provides read/write access to files stored in
[Databricks Unity Catalog volumes](https://docs.databricks.com/en/connect/unity-catalog/volumes.html)
via the [Databricks Files REST API](https://docs.databricks.com/api/workspace/files),
using the official [Databricks SDK for Go](https://github.com/databricks/databricks-sdk-go).

Files and directories are addressed using their volume URI path:

```
/Volumes/<catalog>/<schema>/<volume>/<path>
```

In rclone the leading `/` is omitted, so you configure the root as:

```
Volumes/<catalog>/<schema>/<volume>[/<sub-path>]
```

## Configuration

You will need:

1. Your Databricks workspace URL (e.g. `https://abc-1234.azuredatabricks.net`).
2. Credentials — a [personal access token](https://docs.databricks.com/en/dev-tools/auth/pat.html)
   is the simplest option; see [Authentication](#authentication) for alternatives.

Run `rclone config` and follow the interactive prompts:

```console
rclone config
```

Once configured you can use rclone with paths of the form
`mydb:Volumes/catalog/schema/volume/path`.

### Example usage

List files in a volume:

```console
rclone ls mydb:Volumes/mycatalog/myschema/myvolume
```

Copy a local file to a volume:

```console
rclone copy /path/to/local/file.parquet mydb:Volumes/mycatalog/myschema/myvolume/data/
```

Sync a local directory to a volume:

```console
rclone sync /local/dir mydb:Volumes/mycatalog/myschema/myvolume/destination/
```

## Authentication

The backend delegates authentication entirely to the
[Databricks SDK for Go](https://github.com/databricks/databricks-sdk-go),
which supports the full
[Databricks unified authentication](https://docs.databricks.com/en/dev-tools/auth/unified-auth.html)
flow. Supported methods include:

| Method | How to configure |
|---|---|
| Personal Access Token (PAT) | Set `token` in rclone config **or** `DATABRICKS_TOKEN` env var |
| OAuth M2M (service principal) | Set `DATABRICKS_CLIENT_ID` + `DATABRICKS_CLIENT_SECRET` env vars (leave `token` blank) |
| Azure Managed Identity | Set `ARM_USE_MSI=true` env var (leave `token` blank) |
| Azure Service Principal | Set `ARM_CLIENT_ID`, `ARM_CLIENT_SECRET`, `ARM_TENANT_ID` env vars |
| Databricks CLI profile | Set `DATABRICKS_CONFIG_PROFILE` env var |

When using environment-variable based auth, leave the `token` field blank during
`rclone config`.

## Limitations

- Modification times are supported at second precision (the Files API
  returns millisecond timestamps in directory listings, but HTTP date
  headers returned by `HEAD` have second precision).
- File hashes are not exposed by the Databricks Files API and are
  therefore unavailable.
- Setting modification times is not supported.
- Only empty directories can be deleted with `rclone rmdir`. Use
  `rclone purge` to recursively delete a non-empty directory.
