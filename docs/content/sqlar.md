# {{< icon "fas fa-database" >}} SQLite Archive (SQLAR)

This backend stores files compressed in an [SQLite Archive](https://www.sqlite.org/sqlar.html),
a file container similar to a ZIP archive or Tarball but based on an
SQLite database. It uses the schema defined by the SQLite project.
The entire filesystem is kept in a single `.sqlar` file that can also be
read and written by the standard `sqlite3` command-line tool. Files are
compressed automatically using zlib.

## Configuration

Here is an example of how to make a remote called `myarchive`. First run:

    rclone config

This will guide you through an interactive setup process:

### Connection strings

You can also use SQLAR without a config file using connection strings:

    rclone ls :sqlar,path=/path/to/archive.sqlar:
    rclone copy localdir/ :sqlar,path=/path/to/archive.sqlar:backup/
    rclone cat :sqlar,path=/path/to/archive.sqlar:path/to/file.txt

## Interoperability with sqlite3 CLI

Archives created by rclone are standard SQLAR files. You can use the
`sqlite3` command-line tool (v3.23.0+) to work with them:

```bash
# List archive contents
sqlite3 archive.sqlar -At

# Extract archive
sqlite3 archive.sqlar -Ax

# Add files to archive
sqlite3 archive.sqlar -Ac file1.txt file2.txt

# Query archive with SQL
sqlite3 archive.sqlar "SELECT name, sz FROM sqlar ORDER BY sz DESC LIMIT 10"
```

Archives created by `sqlite3 -Ac` can be read by rclone.

## Backend commands

### vacuum

After deleting files, the archive file doesn't shrink automatically.
Run vacuum to rebuild the database and reclaim unused space:

    rclone backend vacuum remote:

## Compression

Files are compressed using zlib (RFC 1950).

If compression doesn't reduce the size, the file is stored uncompressed.
This is detected automatically on read based on the SQLAR convention:
when `sz` (original size) equals the stored data length, the content
is uncompressed.

You can control the compression level with the `compression_level` option:

- `-1` — zlib's default compression (default)
- `0` — no compression
- `1` — fastest compressed mode
- `9` — best compression

For example, to use maximum compression with a connection string:

    rclone copy localdir/ :sqlar,path=/path/to/archive.sqlar,compression_level=9:

## Limitations

Individual files cannot exceed ~1 GiB. SQLite blobs are limited by the
`SQLITE_MAX_LENGTH` compile-time constant (default 1,000,000,000 bytes).
Attempting to store a larger file will fail immediately with an error.

File content is loaded into memory during read/write when compression is
enabled (the default), so very large files near the limit require
significant RAM.

Only one process should write to an archive at a time.

Modification times have one-second precision.
